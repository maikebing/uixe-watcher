# 交通事件管理接口文档

## 1. 接口列表

### 1.1 交通事件推送信息接收

| 项目 | 内容 |
|------|------|
| **请求路径** | `http://ip:port/**/***/***` |
| **请求方式** | POST |

#### 请求参数

| 参数名 | 参数类型 | 是否必传 | 说明 | 默认值 |
|--------|----------|----------|------|--------|
| recordId | String | 否 | 预警记录id | - |
| eventType | String | 否 | 事件类型：7-拥堵、45-排队溢出、46-排队超限、4-停车超时、trafficFlowStat-车流量 | - |
| LaneNo | String | 否 | 车道号 | - |
| capTime | Date | 否 | 抓拍时间 | - |
| startTime | Date | 否 | 统计事件开始时间 | - |
| period | Int | 否 | 流量值对应的统计时间。单位：分钟 | - |
| periodByMili | Int | 否 | 流量值的毫秒时间，和 Period 一起使用。流量值总时间：period × 60 × 1000 + periodByMili（单位：毫秒） | - |
| maxQueueLen | Float | 否 | 最大排队长度，周期内最大排队长度，单位：米 | - |
| imageList | String | 否 | 图片列表 | - |
| videoList | String | 否 | 视频列表 | - |

#### 返回参数

| 参数名 | 参数类型 | 是否为空 | 说明 |
|--------|----------|----------|------|
| code | Int | 否 | 返回码 |
| message | String | 否 | 返回消息 |

#### 请求成功出参示例

```json
{
    "code": 0,
    "message": "推送成功",
    "timestamp": 1765941141872
}
```
